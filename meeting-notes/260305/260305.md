## 一、核心研究问题

### 问题定位

在**完全未知的环境**中，让机器人**在有限能量内**，同时完成**探索（Exploration）**与**任务处理（Exploitation）**的最优规划。

与现有工作的区别：

- 传统 Organic Search 假设 infomap 已知，只需规划路径
- 本工作的 infomap **完全未知**，需要机器人自己通过感知**在线构建**
- 感知（Perception）＋规划（Planning）**联合优化**，而非先验已知再规划

类比说明：就像整理一个陌生的房间——第一次进去要先环顾一圈（探索），发现重要的东西（如没关的电视、散乱的椅子）后再优先处理，同时逐步更新对这个房间的认知。

---

## 二、关键概念定义

### InfoMap（信息地图）

- 定义：工作空间上的一个**概率密度函数** `I(x, y)`，表示每个位置的**信息重要性**
- 直觉：信息越重要的地方，值越高（如火灾中可能有人的区域 → 高值）
- 机器人一开始不知道 infomap，通过探索不断更新
- 数学本质：类似**热力学方程**（PDE），机器人到达某处会"冷却"该区域，重要物体的地方"热度"高；梯度驱动规划

### 感知（Perception）

- 输入：摄像头图像（俯视，摄像头朝下）
- 目标：将一张局部图像转化为该区域的**局部信息分布**（小的 distribution）
- 输出：归一化的重要性分值，用于更新全局 infomap

### Exploration vs. Exploitation 权衡

- 机器人停留时间与该位置重要性**成正比**（来自 Organic Search 领域的核心假设）
- 关键问题：如何在"继续探索未知区域"和"立刻处理当前发现的高价值目标"之间做最优决策

## 三、技术方案

### 整体框架（三模块）

```
感知模块
  ↓ 局部 distribution
更新模块 → 全局 InfoMap I(x,y)
  ↓ 已知 InfoMap
规划模块 → 最优轨迹
```

#### 模块1：感知（Perception）

1. 机器人飞到某个 grid cell，用朝下摄像头拍一张照片
2. 神经网络处理该图像：
   - 输入：grid cell 图像
   - 输出：该 cell 中目标物体的数量（或归一化重要性分值）
3. 简化假设：**先验知道目标类型**（如"人"、"方块"），神经网络只需做目标计数/分割，无需识别
4. 初步实现：图像分割，统计方块数量 → 转化为重要性分值

#### 模块2：更新（Update）

1. 将局部 distribution（单次感知结果）更新到全局 infomap
2. infomap 是一个离散化的二维网格，每个 grid cell 对应一个标量值
3. 视野（FOV）是连续的范围，但空间离散化后，每个 grid cell 只对应一个数值点
4. 更新方式：类似贝叶斯更新，也可参考热力学方程的梯度驱动方式

#### 模块3：规划（Planning）

- 发言人已有现成代码，暂不需要其他人实现
- 接口设计：规划模块只需要一个 PDF 函数 `f(x, y) → 概率`
- 规划本质：在有限能量约束下，最大化"已覆盖信息"的积分
- 数学形式：PDE（偏微分方程）求解，类似之前发表的 TRO 论文的梯度方法

### 离散化设计

- 工作空间被划分为 N×N 网格（grid）
- 每个 grid cell 是离散的空间点，对应 infomap 中的一个数值
- 视野（FOV）是连续范围，一次拍照可覆盖多个 grid cell
- 关键区分：**空间 grid** ≠ **视野**，视野是连续的，grid 是离散化的

### 能量模型

- move cost：移动到相邻 grid cell 的能耗
- action/process cost：在某处完成任务（如处理目标物）的能耗
- 初始简化：两者能耗相同
- 不计算感知能耗（感知视为免费）
- 总能量已知，用于规划约束

## 四、实验场景设计

### 基本仿真设置

- **维度**：2D 平面（先做 2D，后续扩展至 3D）
- **场景**：已知边界的有限工作空间（如房间俯视图）
- **机器人**：俯视无人机，摄像头朝下
- **目标物**：随机散布的**方块**（统一颜色，方便图像分割）
- **能量限制**：机器人每次出发只能走有限步，需返回充电

### 数据集构建

- 自己制作：在 2D 地图上随机撒方块，用踩点采样即可
- 不依赖开源数据集（目前无类似开源数据）

### 测试流程（单次 Episode）

1. 机器人从随机（或固定）起点出发
2. 每到一个 grid cell，拍照获取局部 distribution
3. 更新 infomap
4. 规划下一步行动（探索新区域 or 处理已知高价值区域）
5. 能量耗尽后返回充电，下次出发时已有先验知识

## 五、学术定位与创新点

### 与现有工作的区别

| 方面       | 现有 Organic Search 工作         | 本工作                               |
| ---------- | -------------------------------- | ------------------------------------ |
| InfoMap    | 已知（oracle）                   | 未知，在线构建                       |
| 感知模型   | Delta function 或 Gauss function | 实际图像感知（方形硬边界 footprint） |
| 边界平滑性 | 需要 T-degree 处理               | 不 care T-degree（直接离散化）       |
| 探索       | 不需要                           | 核心问题                             |

### 创新点

1. **将感知（perception）引入 Organic Search**，填补了该领域不考虑实际 footprint 的空白
2. **在未知 infomap 的情况下**，通过边感知边规划，达到接近 oracle（已知 infomap）的效果
3. footprint 使用精确的方形视野（而非高斯或 Delta），更符合实际传感器模型

### Baseline 设计

- **Oracle baseline**：假设 infomap 完全已知，直接规划（理论上界）
- **目标**：本方法在探索约束下，效果能达到 oracle 的**同一数量级**即可（不要求完全相同）
- 意义：额外付出了探索代价，但效果不显著下降

## 六、应用场景讨论

讨论了多种实际应用：

1. **火灾搜救**：无人机在未知建筑中搜索被困人员，重要性 = 发现人的概率，需在有限电量内尽可能搜索更多区域
2. **仓储物流**：机器人在仓库中分配任务，高价值货物 vs. 高频低价值货物之间的搬运效率权衡（类似旅行商问题的扩展）
3. **野生动物追踪**：在森林中搜索珍稀动物，足迹多的区域概率高 → 重要性高

## 七、未来扩展方向

1. **动态 infomap**：目标物体可以移动（如人），每次探索后更新概率分布
2. **多智能体（Multi-Agent）**：多个机器人共享探索信息，通过通讯协作
   - 可用两层神经网络：底层各 agent 探索，顶层 coordinator 汇总估测全局 infomap
   - 发言人认为当前作业阶段先不加入此功能，留作扩展
3. **3D 场景**：从 2D 扩展到多层建筑（如三层楼的搜救），充电策略更复杂
4. **可变地图**：加入障碍物动态变化的场景
